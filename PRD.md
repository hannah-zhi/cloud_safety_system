# 储能云端安全预警系统 - 安全预警页面 PRD

## 1. 项目背景

集团旗下管理 100+ 储能场站，场站分布广、设备类型多、告警链路长。现有运营人员需要在云端快速识别通讯异常、停机场站、高风险场站和单站内部子系统风险，减少人工巡检和跨系统查询成本。

本页面命名为“安全预警”，在左侧导航中作为独立菜单呈现。页面首页参考“场站列表”视图，以卡片矩阵展示各场站当前安全和通讯状态；点击任意场站后进入单站详情，视觉风格和指标布局参考“安全诊断”页面。

## 2. 产品目标

1. 在首页集中呈现 100+ 场站的运行安全态势，支持运营人员快速筛选高风险和通讯异常场站。
2. 将场站风险、通讯状态、功率、电量、SOC 等核心指标压缩到卡片中，便于横向对比。
3. 支持从集团场站总览钻取到单站详情，查看 SOS 安全指数、风险等级分布、子系统风险和告警明细。
4. 通过深色大屏风格、蓝色高亮、红黄绿风险分层，匹配现有远景产品 UI 语义。

## 3. 用户角色

- 集团安全运营管理员：关注所有场站的安全等级、通讯情况和待处置告警。
- 区域运维负责人：按站点名称、状态筛选辖区内重点场站，并进入详情定位风险来源。
- 单站值班人员：查看本站 SOS 指数变化、子系统异常和风险描述，辅助现场处置。

## 4. 信息架构

### 4.1 全局框架

- 顶部栏：系统名称、当前页面名称、日期时间、登录用户。
- 左侧栏：场站概览、风险预警、安全预警、健康评估、运行评价、系统管理，其中“安全预警”高亮。
- 主内容区：
  - 首页：场站列表。
  - 单站详情：安全诊断详情。

### 4.2 首页 - 场站列表

首页展示集团全部场站，默认显示 110 个场站卡片。

核心模块：

- 顶部页签：已选总数、场站列表、设备列表。
- 搜索框：按场站编号和名称模糊搜索。
- 状态统计筛选：
  - 通讯正常。
  - 部分通讯中断。
  - 通讯中断。
  - 高风险。
  - 中风险。
  - 低风险。
  - 健康。
- 排序控件：
  - 按场站名称。
  - 按 SOS 指数。
  - 按风险等级。
  - 按 SOC。
- 视图控件：卡片矩阵布局。
- 场站卡片：
  - 场站状态标签：待机 / 运行 / 停机。
  - 风险标签：高风险 / 中风险 / 低风险 / 健康。
  - 场站编号与名称。
  - 场站通讯状态。
  - 额定功率。
  - 场站有功功率。
  - 剩余电量。
  - 场站 SOC。
  - SOS 指数。

交互：

- 点击状态统计筛选卡片，筛选场站列表。
- 输入搜索关键词，实时过滤。
- 切换排序，列表重新排序。
- 点击场站卡片，进入单站安全详情。

### 4.3 单站详情 - 安全诊断

详情页基于所选场站展示单站安全诊断大屏。

核心模块：

- 面包屑/返回：从单站详情返回场站列表。
- 场站概览条：场站名称、通讯状态、风险等级、SOS 指数、SOC、当前告警数量。
- 全站当前 SOS 安全指数：
  - 半圆仪表盘。
  - 当前风险等级提示。
- 全站 SOS 安全指数历史趋势：
  - 最近 7 天折线。
  - 高风险、中风险阈值参考线。
  - 时间范围切换：7 天 / 15 天 / 30 天。
- 子系统风险等级分布：
  - 环形图展示高风险、中风险、低风险、健康数量。
- 各子系统 SOS 安全指数：
  - 多子系统柱状图。
  - 支持按分数高低切换排序。
- 设备部件 SOS 指数分析：
  - 以箱线图风格呈现电池、电气、环控、消防四类部件风险区间。
- SOS 安全指数详情清单：
  - 子系统。
  - 当前 SOS 指数。
  - 更新日期。
  - 风险描述。
  - 处置建议。

交互：

- 点击返回按钮回到首页，保留搜索和筛选状态。
- 点击时间范围按钮，趋势图数据刷新。
- 点击子系统排序按钮，柱状图排序切换。
- 点击表格行，高亮对应风险项。

## 5. 风险分级规则

- 高风险：SOS < 60，颜色红色。
- 中风险：60 <= SOS < 80，颜色黄色。
- 低风险：80 <= SOS < 90，颜色绿色。
- 健康：SOS >= 90，颜色蓝色。

通讯状态：

- 通讯正常：绿色。
- 部分通讯中断：黄色。
- 通讯中断：红色。

## 6. 视觉与交互规范

- 整体风格：深色工业大屏、低饱和背景、高对比信息层级。
- 背景色：近黑色主背景，深灰卡片。
- 主色：科技蓝，用于高亮导航、选中态、图表主线。
- 风险色：
  - 红色：高风险、通讯中断、停机。
  - 黄色：中风险、部分通讯中断。
  - 绿色：低风险、通讯正常。
  - 蓝色：健康和系统主指标。
- 卡片边框：顶部细线强调，模拟参考图中场站卡片样式。
- 圆角：不超过 8px。
- 字体：系统默认无衬线字体，中文优先。
- 动效：筛选、进入详情、按钮 hover 使用轻量过渡，不影响大屏稳定性。

## 7. 数据要求

当前开发阶段使用前端模拟数据：

- 场站数量：110。
- 每个场站包含编号、名称、通讯状态、运行状态、风险等级、额定功率、有功功率、剩余电量、SOC、SOS 指数。
- 单站详情根据所选场站动态生成趋势、子系统、部件分析和告警表格。

后续接口建议：

- `GET /api/safety-warning/stations`
- `GET /api/safety-warning/stations/{stationId}/overview`
- `GET /api/safety-warning/stations/{stationId}/trend?range=7d`
- `GET /api/safety-warning/stations/{stationId}/subsystems`
- `GET /api/safety-warning/stations/{stationId}/alerts`

## 8. 验收标准

1. 左侧导航存在“安全预警”并处于选中态。
2. 首页展示 100+ 场站卡片，支持搜索、筛选、排序。
3. 场站卡片点击后进入单站详情。
4. 单站详情包含仪表盘、趋势图、环形图、子系统柱状图、部件分析和详情清单。
5. 返回按钮可回到首页。
6. 视觉风格与参考图一致：深色底、蓝色选中态、红黄绿蓝风险分层。
7. 页面可在本地浏览器直接访问，无需构建步骤。
